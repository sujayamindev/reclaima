Why it exists

Why Receipta exists

The problem

Most people have no idea when their warranties expire. Receipts get lost, warranty cards go unfiled, and return windows close before anyone notices. When something breaks, the process of finding proof of purchase, identifying warranty coverage, and drafting a claim is manual, scattered, and frustrating enough that most people don't bother.

What exists today

The tools that exist today solve half the problem at best. Business expense apps have strong OCR but treat receipts purely as financial records — no warranty tracking, no return window, no claim support. Dedicated warranty trackers let you log products and set reminders, but rely on manual entry, treat each receipt as a single item, and stop at the notification. None of them generate anything you can actually submit to a retailer or manufacturer.

What Receipta does differently

Receipta covers the full purchase-to-claim lifecycle in one place. Receipts are scanned using AWS Textract with a Claude Haiku cleanup layer that handles noisy, locally formatted receipts. Every line item is extracted individually, so a single receipt with five products creates five independently tracked entries, each with its own warranty period, return window, and reminder schedule. When a product fails, Receipta generates a structured PDF warranty claim document — with purchase details, issue description, and defect photos attached — ready to submit directly to a retailer or manufacturer. How it works

Four steps.
Zero lost warranties.

1

Capture

Snap a photo, pick from gallery, or upload a PDF. Front and back both sides supported. Up to 20 MB per upload.

2

Extract

AWS Textract reads every line. Claude Haiku fixes OCR noise — garbled names, transposed digits, broken totals.

3

Track

Each line item gets its own warranty and return countdown with configurable FCM push reminders.

4

Claim

One tap generates a claim-ready PDF with purchase details, defect photos, and retailer contact.

FCM · APScheduler
adf_scanner

Claim PDF generation

Generates a structured, print-ready PDF with purchase details, product images (Brave Search API), warranty dates, and store contact.

Backend · Brave Search
layers

Dual-sided receipt scanning

Both front and back receipt images are processed separately through the OCR pipeline. Results are merged — front image provides header fields (store, date, total), line items from both sides are combined into a single structured output. Both images are included in the generated claim PDF.

Front & back · OCR
track_changes

Claim lifecycle management

Every warranty claim has a full status lifecycle: DRAFT → SUBMITTED → IN_PROGRESS → RESOLVED / DENIED. On resolution, three outcomes are supported — REFUNDED (item archived), REPAIRED (item stays active, user prompted to update warranty dates), REPLACED (original archived, replacement item created and linked). Replacement chains are tracked via linked records.

DRAFT → RESOLVED
image_search

Product image auto-fetch

After OCR processing, Receipta automatically fetches a product image for each line item via Brave Image Search. Results are ranked by domain trust — official brand sites and major retailers are preferred over marketplaces and aggregators. Images are stored and displayed on the product detail screen.

Brave Search · auto
call_split

Quantity-aware line item expansion

If a scanned receipt contains a line item with quantity greater than one, Receipta creates that many individual records — each independently tracked with its own warranty period, return window, and reminder schedule.

Per-unit tracking
tune

Per-item notification controls

Each product has its own notification configuration — independent lead day overrides for warranty and return reminders, and individual enable/disable toggles. Tapping a notification deep links directly to the specific product screen.

Deep links · FCM

SYSTEM_PROMPT = """You are an OCR cleanup assistant for retail receipts.
Given raw AWS Textract block output, return a single JSON object with:
  vendor, invoice_no, date (ISO 8601), currency, total, line_items[].
Each line item: name, qty, unit_price, category.
Fix OCR errors (garbled characters, transposed digits).
Infer missing fields where context is clear. Return only valid JSON."""

async def cleanup_ocr(self, blocks: list[dict]) -> ReceiptExtractResponse:
    raw_text = _blocks_to_text(blocks)
    response = await self.client.invoke_model(
        modelId="anthropic.claude-haiku-20240307-v1:0",
        body={"system": SYSTEM_PROMPT, "messages": [{"role": "user", "content": raw_text}]},
    )
    return ReceiptExtractResponse.model_validate_json(response["content"][0]["text"])
